# 决定

笔记正文里的双链优先指向概念页(`[[concepts/某概念]]`),笔记之间不直接互链——两篇笔记通过共同概念在图谱里相连。例外:两篇确实是同一论证的上下篇时可直链。概念页采用懒加载:录入时只写链接(幽灵节点零成本),第二次命中才建页,延伸区只在用户要求或每周 GC 时补。完整规则见 CLAUDE.md「概念层」一节。

# 为什么

- 2026-07-04 用户看到笔记互链的效果后明确反馈:内嵌互链读起来是"强行把另一篇笔记塞进来",且图谱只长出文档引用网,想看到的是概念的连接——这是对前一条约定(内嵌互链)的升级
- 概念页让图谱节点=思考单位;笔记互链让图谱节点=文件,粒度不对
- 懒加载是用户对 token 成本的直接顾虑("每输入一篇要做的事太多了"),经确认采纳:重活全部挪到"用户在场且想要"的时刻

# 何时失效

用户表示更喜欢别的链接风格或概念层维护太重时,按「人类有否决权」直接修正。
