# 决定

`工作台.md` 这半个月密集迭代掉了一批曾经存在、后来被用户明确否决的元素。CLAUDE.md「工作台」一节只保留"现在长什么样、不要恢复成什么样"的精简清单,这些改动各自的时间点和用户原话搬到这里存档,以后如果要改工作台又想知道"这个东西当初为什么被去掉"可以来查,不用每次会话都无条件加载这段历史。

  - 三次修:随机重逢下滑再上滑不会像问候语/名画那样重新渐现——根源是 Obsidian 阅读视图对长笔记分段懒渲染,画作那块滚出视口会被卸载、滚回来重新创建 DOM 从而重播"刚创建就播一次"的入场动画,随机重逢这块没被卸载重建,动画没有重新播放的触发时机。改成用 IntersectionObserver 直接盯可视区域交叉,不依赖 DOM 是否被重建,每次滚进视口强制切一次 `wb-scroll-reveal` class 重播动画(详见 工作台.md reunion 代码块末尾 + workbench.css 对应注释)

# 为什么统一收在这一篇

这些都是"删掉/否决过的东西",清单本身很短,但每条如果都展开写清楚时间+原话+理由,对当前操作没有增量帮助——AI 只需要知道"不要加回来",不需要每次都重新理解"当初为什么加回来的冲动会出现"。放在这里按需查阅,CLAUDE.md 正文只放一句话清单。
